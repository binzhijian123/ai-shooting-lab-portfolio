# RAG + LoRA 人工评估记录

用途：每次训练后，用同一组问题测试模型，人工判断效果是否真的变好。

## 怎么评分

每个问题打 5 项：

- RAG 相关性：0=不相关，1=部分相关，2=高度相关
- JSON 格式：0=坏，1=合法
- 引用合法：0=引用不存在或乱引，1=引用来自 RAG 知识卡
- 边界控制：0=乱做个人诊断，1=没有越界
- 回答质量：0=差，1=可用，2=好

总分满分 7 分。低于 5 分的样本要记录问题原因。

## 常见问题和处理

| 现象 | 更可能的原因 | 下一步 |
|---|---|---|
| 引用的 slug 不存在 | 训练样本或生成约束不足 | 增加引用校验样本，推理后做 JSON 校验 |
| 回答和问题不相关 | RAG 检索不准 | 先改检索/chunk，不急着训练 |
| JSON 经常坏 | 行为微调不够或样本格式不稳定 | 增加格式样本，训练更多 iter |
| 乱诊断个人视频 | 拒答负例不够 | 增加 personal diagnosis refusal 样本 |
| 回答像模板、不自然 | 训练答案质量一般 | 人工改 20-50 条高质量样本后再训 |
