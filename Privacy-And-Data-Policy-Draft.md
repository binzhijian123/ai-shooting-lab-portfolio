# Privacy and Data Policy Draft

Updated: 2026-06-14

## 当前数据状态

当前实现是本地 Web 原型：

- 上传视频保存到本机 `data/uploads/`。
- 训练记忆保存到本机 SQLite：`data/shooting_lab.sqlite`。
- DeepSeek 请求只接收结构化 evidence packet，不应包含原始视频、base64 视频、服务器路径或完整转写。
- 没有登录、账号、云端同步或组织权限。

## 默认规则

真实校队视频默认不得：

- 公开展示。
- 外部分发。
- 上传云端保存。
- 用于训练模型。
- 放入公开作品集。
- 分享给第三方服务。

除非用户明确提供并授权，系统不得读取、上传或处理真实校队视频。

## 本地 SQLite

本地 SQLite 可保存：

- `session_id`
- 保存时间
- 本地标题
- `user_id` 默认值
- 视角、帧率、置信度
- `metrics_json`
- `signals_json`
- `report_json`
- `evidence_json`
- 用户反馈

本地 SQLite 不应保存：

- API key。
- 账号密码。
- 未授权个人身份信息。
- 云端访问 token。
- 原始视频二进制。

## 上传视频

当前上传视频会保存到本机 `data/uploads/` 供 adapter 使用。当前本地原型已支持删除当前进程登记的 `upload_id` 对应原始上传文件，也支持列出并逐个删除文件名符合 `upload_*` 格式的受控历史上传文件，还支持按保留天数 dry-run 预览和手动执行清理；删除路径限制在 `data/uploads/`。产品化前仍必须补充：

- 保存期限。
- 后台自动清理策略。
- 样例视频和真实校队视频的目录隔离。
- 授权状态记录。

## 未来云端

登录、账号、云端同步和跨设备恢复是后续目标，不是当前已实现事实。

设计云端前必须确认：

- 部署服务商。
- 数据存储区域。
- 用户身份和组织权限模型。
- 视频是否上传云端。
- 原始视频保存期限。
- 谁能查看球员报告。
- 删除和导出机制。
- 是否允许匿名化内部迭代。

## 授权

每段真实校队视频至少需要记录：

- 提供者。
- 球员或监护人授权状态。
- 使用范围：仅本地分析、内部规则迭代、展示、模型训练。
- 保存期限。
- 是否可脱敏展示。
- 是否可删除和撤回。

默认授权范围是“仅本地分析”，不是内部训练或公开展示。

当前本地原型新增了 metadata-only 授权样例 readiness gate：`scripts/authorized-sample-readiness-smoke.mjs`。该 gate 只检查 manifest/fixture 元数据，不读取、上传、解码或分析真实视频文件。真实或代表性样例进入 Phase 1 本地验收前，必须至少满足：

- `authorization.status=authorized`。
- `authorization.scope` 包含 `local_analysis` 和 `local_acceptance_test`。
- `authorization.scope` 不包含公开展示、外部分发、云端保存或模型训练。
- 主体授权记录为 documented，并记录授权 authority 和日期。
- retention 为 `local_only`，包含原始视频删除天数和 review 日期。
- privacy 显式禁止公开展示、外部分发、云端保存和模型训练。
- `expected_use.diagnosis_confidence` 只能是代表性验证或需要人工复核，不能写成稳定球员诊断。

## 删除

第一版必须提供或至少定义：

- 删除单次上传视频。
- 删除单次 session。
- 删除某个本地用户的全部 SQLite 记录。
- 删除导出的报告或样例包。

当前本地原型已支持：

- 删除当前进程登记的单次上传原始文件。
- 删除受控历史上传文件名对应的本地上传文件。
- 保留期清理的 dry-run 预览和手动执行。
- 删除单条本地 SQLite session。
- 删除某个本地 `user_id` 的全部本地 SQLite sessions。

当前删除边界：

- 删除本地 SQLite session 不会删除上传视频文件。
- 删除上传文件只覆盖 `data/uploads/` 下受控文件，不覆盖外部文件路径。
- 当前没有账号、组织权限、授权撤回、云端导出或云端删除流程。
- 当前没有原始视频导出能力。

删除后不得继续在报告、样例、公开页面或训练集中引用该视频。

## 内部迭代用途

允许在明确授权后使用的内容：

- 脱敏后的结构化指标。
- 缺失证据类型统计。
- 用户确认的动作标签。
- 不含原始视频的人体关键点时间序列。

仍需谨慎的内容：

- 原始视频。
- 可识别球员身份的截图。
- 队名、学校、号码、人脸、声音。
- 训练地点和时间。

## 第三方服务

- DeepSeek 或其他大模型服务不得接收原始视频。
- 外部模型、云存储或付费 API 不能成为硬依赖，除非用户确认。
- API key 只允许在后端环境变量或本地 `.env`，不得进入前端 bundle、HTML、日志或长期记忆。

## Phase 0.5 结论

当前隐私边界适合本地原型和授权样例验证，不适合直接处理、保存或展示真实校队视频的生产流程。本地 JSON 导出只覆盖 SQLite session、记忆摘要和上传文件清单元数据，不导出原始视频字节。Phase 7 前不得把云端同步、账号登录、云端导出/删除或真实队伍数据治理写成已完成能力。
